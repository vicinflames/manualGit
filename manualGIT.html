Manual Básico de GIT

- En primer lugar si queremos utilizar las funciones que nos ofrece GIT, podremos instalarlo con la terminal, introduciendo el comando: apt-get install git.

- Una vez ya tengamos disponible el programa, crearemos una carpeta donde realizaremos todas las tareas que requerirán el uso de git. Luego dentro de la carpeta introduciremos el comando git init que no inicializara git dentro de la carpeta creada.

- Cuando tengamos algunos archivos creados dentro de la carpeta inicializada, puede que nos interese ver el estado en el que están, por lo tanto usaremos el comando git status.

- Luego en el momento de que tengamos archivos terminados, antes de iniciar el seguimiento de los cambios realizados, primero tendremos que añadirlo a la zona de espera utilizando git add. También podemos generalizar para no tener que añadir archivo uno por uno, por ejemplo: git add '*.html'.

- Al haber realizado un git add los archivos estarán en el 'Staging Area' (área de ensayo) y aún no los tendremos en nuestro repositorio. Para guardar nuestros cambios escalonados pondremos el comando git commit -m con un mensaje que describa lo que hemos cambiado dentro de comillas.

- Una vez realizado unas cuantas confirmaciones, podremos navegar para ver lo que hemos cambiado. Es como un diario que recuerda todos los cambios que hemos comprometidos hasta el momento en su respectivo orden. Para ver eso ejecutaremos el comando git log.

- Si queremos tener un nuevo repositorio en GitHub usaremos Git Try. Y para usarlo, tendremos que añadir un repositorio remoto, con un comando que empezara por git remote, un nombre remoto y una URL del repositorio. Por ejemplo: git remote add origen https://github.com/try-git/try_git.git.

- Cuando tu proyecto se encuentra en un estado que quieres compartir, tienes que enviarlo a un repositorio remoto. El comando que te permite hacer esto es: git push [alias] [branch].

- Y para comprobar si han hecho cambios en nuestro repositorio GitHub es el comando: git pull. Un ejemplo claro es el siguiente: git pull origin master.

- Otro comando útil que tiene git es que puedes ver las modificaciones que ha habido respecto a la ultima vez. El comando que permite hacer esto es: git diff --staged.

Para retirar archivos que se han añadido en git se usa: git reset [file].

Pero git reset no quita el archivo del todo, simplemente no lo podemos visualizar. Por eso los archivos se pueden cambiar de nuevo a la forma en que estaban en la ultima confirmación mediante el comando: git checkout -- [destino].

También podemos crear copias locales del repositorio que estamos ejecutando con git clone /path/to/repository y si utilizas un servidor remoto, ejecutamos git clone username@host:/path/to/repository

Con git checkout podemos crear o cambiar a otra rama añadiendo -b y con git branch podemo ver o borrar una rama utilizando el parámetro -d.

Git rm elimina archivos desde el árbol de trabajo y del índice.

Podemos fusionar distintas ramas en una con git merge y el nombre de la rama.

En caso de que queramos ver los cambios que se han hecho con el tiempo en una determinada rama usaremos el comando git k y se nos abrirá una ventana para poderlo en modo gráfico.
